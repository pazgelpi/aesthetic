# Aesthetic IQ — Setup Guide

## 1. Supabase Project

1. Go to [supabase.com](https://supabase.com) → New project
2. Note your **Project URL**, **anon key**, and **service role key** from Project Settings → API

## 2. Environment Variables

Copy `.env.example` to `.env.local` and fill in:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
ANTHROPIC_API_KEY=sk-ant-...
TWILIO_ACCOUNT_SID=AC...               # Optional for WhatsApp
TWILIO_AUTH_TOKEN=...                   # Optional for WhatsApp
TWILIO_WHATSAPP_FROM=+1415...          # Your Twilio WhatsApp number
NEXT_PUBLIC_APP_URL=http://localhost:3000
CRON_SECRET=any-random-secret-string
```

## 3. Database Migrations

In Supabase Dashboard → SQL Editor, run in order:

1. `supabase/migrations/20260502000001_initial_schema.sql`
2. `supabase/migrations/20260502000002_rls_policies.sql`
3. `supabase/seed.sql` (templates + protocols)

## 4. Supabase Storage Buckets

Create two public buckets in Storage:
- `treatment-photos` — for patient photos
- `clinic-documents` — for knowledge base files

For both, set **Public bucket = ON**.

## 5. Auth Configuration

In Supabase → Authentication → URL Configuration:
- **Site URL**: `http://localhost:3000` (or your production URL)
- **Redirect URLs**: Add `http://localhost:3000/auth/callback`

In Authentication → Email → Enable "Email OTP" (magic link).

## 6. Run Locally

```bash
npm install
npm run dev
```

Open http://localhost:3000 → it will redirect to login → enter your email → check inbox → click magic link.

## 7. First Login Flow

1. Log in with magic link
2. You're redirected to **Onboarding** → create your clinic
3. You're redirected to **Settings** → 6-step wizard:
   - Step 1: Branding (name, city, color)
   - Step 2: Tone (pronoun, formality, emoji)
   - Step 3: Voice samples (3 examples of how you write)
   - Step 4: Message signature
   - Step 5: Protocols (toxin + filler settings)
   - Step 6: Knowledge base (optional PDF upload)
4. After wizard → Dashboard

## 8. WhatsApp Setup (Twilio)

1. Create Twilio account → WhatsApp Sandbox or Business Number
2. Set webhook URL in Twilio to `https://your-domain.com/api/webhooks/twilio/inbound`
3. Add Twilio credentials to `.env.local`

Without Twilio credentials, messages are simulated (logged to console only).

## 9. Cron Job (Vercel)

The `vercel.json` configures a cron at `* * * * *` (every minute) hitting `/api/cron/process-messages`.

For local testing, call it manually:
```bash
curl -H "Authorization: Bearer your-cron-secret" http://localhost:3000/api/cron/process-messages
```

## 10. Patient Portal

The portal runs at `/portal/[token]` — no auth required. Tokens are:
- Generated by the clinic when requesting post photos
- Sent to patients via WhatsApp or copy-paste
- Valid for 7 days
- 3 tabs: Mi plan, Mi rutina (protocol), Mis fotos (camera capture)

## Architecture

```
app/
  (app)/          # Authenticated clinic dashboard
    dashboard/    # Traffic light overview + KPIs
    patients/     # Patient CRUD + detail
    treatments/   # Treatment CRUD + photo sessions
    messages/     # WhatsApp message queue
    settings/     # Clinic wizard
  auth/           # Magic link login + callback
  portal/[token]/ # Patient portal (no auth)
  api/
    treatments/schedule-messages/  # Schedules 5 WA messages
    cron/process-messages/          # Cron: generates + sends
    webhooks/twilio/inbound/        # Handles patient replies
    comparisons/generate/           # AI before/after analysis
    photo-sessions/                 # Photo upload + normalization
    recommendations/generate/       # AI patient recommendations
    clinic-documents/extract/       # PDF text extraction
    portal/[token]/                 # Portal data endpoint
```
